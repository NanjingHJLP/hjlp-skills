# 搜索策略参考手册

本文件提供搜索查询的构造技巧和策略，帮助在广度搜索阶段获得更高质量的结果。

---

## 查询构造原则

### 基本原则

1. **具体优于宽泛**：`Rust 异步编程 tokio 性能` 优于 `Rust 编程`
2. **加时间限定**：追加年份（如 `2026`）获取最新信息
3. **用对比结构**：`A vs B` 格式能触发对比类内容
4. **混合中英文**：技术话题用英文搜索往往结果更丰富

### 查询模板

| 研究目的 | 查询模板 | 示例 |
|----------|----------|------|
| 了解定义 | `[主题] 是什么 定义 原理` | `量子计算 是什么 原理 入门` |
| 获取最新信息 | `[主题] [年份] 最新 动态` | `大模型 2026 最新 进展` |
| 对比分析 | `[A] vs [B] 对比 优缺点` | `PostgreSQL vs MySQL 2026 对比` |
| 查找案例 | `[主题] 应用案例 实践 经验` | `微服务架构 企业案例 实践` |
| 了解趋势 | `[主题] 趋势 预测 [年份]` | `AI 行业 趋势 预测 2026` |
| 查找数据 | `[主题] 数据 统计 报告 [年份]` | `中国 SaaS 市场 数据 报告 2026` |
| 查找教程 | `[主题] 教程 入门 指南` | `Kubernetes 入门 教程 指南` |
| 查找评价 | `[主题] 评测 评价 体验` | `Claude vs GPT 评测 对比 2026` |

## 搜索策略

### 策略一：维度交叉搜索

对同一主题从不同维度分别搜索，确保覆盖面。

```text
主题：评估 Rust 是否适合我们的项目

维度 1（技术特性）：Rust 语言 特点 优势 内存安全
维度 2（对比评估）：Rust vs Go vs C++ 性能 开发效率 对比
维度 3（实践经验）：Rust 企业 生产环境 案例 经验
维度 4（学习成本）：Rust 学习曲线 团队 迁移 成本
```

### 策略二：漏斗式搜索

先广后窄，逐步聚焦。

```text
第一轮（广度）：Rust 编程语言 2026
第二轮（聚焦）：Rust async 异步编程 性能 基准测试
第三轮（深入）：tokio vs async-std 对比 生产环境 选择
```

### 策略三：正反搜索

主动搜索正面和反面观点，避免信息茧房。

```text
正面：Rust 优势 为什么选择 Rust
反面：Rust 缺点 不适合 Rust 的场景 Rust 批评
```

### 策略四：多语言搜索

对于技术和国际话题，中英文各搜一轮。

```text
中文：Rust 语言 2026 国内 应用 案例
英文：Rust programming language 2026 production use cases
```

- [主题] 工作原理 机制

## 搜索结果不足时的应对

| 问题 | 应对策略 |
|------|----------|
| 结果太少 | 放宽关键词、去掉时间限定、换同义词 |
| 结果太旧 | 加年份限定、搜索「最新」「2026」 |
| 结果太浅 | 加「深度分析」「白皮书」「研究报告」 |
| 结果同质化 | 换搜索引擎、换语言、搜索学术来源 |
| 找不到数据 | 搜索「报告」「统计」「数据」「survey」「benchmark」 |
| 找不到反面观点 | 搜索「缺点」「批评」「问题」「limitations」「criticism」 |

## 搜索质量自检

完成搜索后，检查以下项目：

- [ ] 每个子查询都独立搜索了（不是只搜了一次）
- [ ] 来源类型多样（不全来自同一个网站）
- [ ] 包含了正面和反面信息
- [ ] 时效性信息有近期来源
- [ ] 技术话题包含了英文来源
- [ ] 总结果数量 >= 15 条
