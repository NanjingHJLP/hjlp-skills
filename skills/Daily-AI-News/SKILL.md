# 每日 AI 新闻简报

聚合多来源 AI 新闻 → 筛选去重 → 分类呈现。

## 工作流程

```
收集 → 筛选 → 分类 → 输出
```

### 1. 收集信息

**直接抓取**：从 3-5 个主要新闻源获取内容（来源列表见 [news_sources.md](references/news_sources.md)）。

**搜索补充**：用网络搜索发现额外新闻，动态替换日期。运行 `scripts/generate_queries.py` 生成当日查询，或参考 [search_queries.md](references/search_queries.md) 手动构建。

搜索最佳实践：
- 始终使用当前/昨天日期过滤
- 跨不同类别执行 2-3 个查询
- 每个查询取前 10-15 个结果
- 优先最近 24-48 小时的内容

**获取全文**：对最相关的 10-15 篇文章，获取完整内容以确保摘要准确。

### 2. 筛选内容

**保留**：24-48h 内的重大公告、产品/模型发布、研究突破、行业发展（融资/合作/法规/收购）、主要公司更新。

**移除**：重复内容（保留最全面版本，注明替代来源）、次要更新/营销、>3天旧内容、非AI内容。

### 3. 分类

| 类别 | 内容 |
|------|------|
| 🔥 重大公告 | 产品发布、模型发布、重大公司公告 |
| 🔬 研究与论文 | 学术突破、新论文、新技术、基准成就 |
| 💰 行业与商业 | 融资、并购、合作、市场趋势 |
| 🛠️ 工具与应用 | 新工具/框架、开源发布、开发者资源 |
| 🌍 政策与伦理 | 法规政策、安全伦理、社会影响 |

### 4. 输出

使用标准模板输出（完整模板见 [output_templates.md](references/output_templates.md)）：

```
# 📰 每日 AI 新闻简报
日期 | 来源数 | 覆盖范围

## [类别]
### [标题]
摘要：一句话概述
要点：2-3 个重要细节
影响：为什么重要
来源 + 链接

## 🎯 关键要点
1-3 条最重要的发展
```

## 自定义

提供简报后，询问用户是否需要调整：

- **关注领域**：仅研究 / 产品工具 / 行业融资 / 特定公司
- **详细程度**：简要（仅标题）→ 标准（默认）→ 深入（含分析）
- **时间范围**：24h（默认）/ 3天 / 一周
- **组织方式**：按类别（默认）/ 按时间 / 按公司 / 按重要性

## 后续互动

| 用户请求 | 操作 |
|----------|------|
| "告诉我更多关于 [X]" | 获取全文，提供详细摘要+分析 |
| "专家怎么看 [Y]" | 搜索专家意见和分析文章 |
| "找类似 [Z] 的新闻" | 搜索相关主题，提供比较 |
| "只看研究论文" | 过滤重组，排除非研究内容 |

## 错误处理

- 某来源获取失败 → 跳过，尝试下一个
- 搜索无结果 → 扩大日期范围或换查询词
- 结果过多 → 提高重要性阈值
- 付费墙内容 → 用可用摘录，注明限制

## 质量检查

- 链接有效可访问
- 无跨类别重复
- 内容有时间戳（优先今天）
- 摘要准确（非虚构）
- 来源多样化
