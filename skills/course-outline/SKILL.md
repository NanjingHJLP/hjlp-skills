# 课程大纲生成 Skill

当用户需要把零散资料（文档、链接、笔记、已有讲义）整理成**结构化课程大纲**时，按本流程执行。输出统一为「课程大纲 + 章节 + 知识点 + 练习建议」格式，便于后续按章学习或二次加工。

## 何时使用（触发语）

- 「整理成一门课」「做课程大纲」「把这份材料拆成章节」
- 「设计学习路径」「讲义结构化」「根据笔记/链接生成课程结构」
- 「把这篇文档/链接变成课程」「给这份内容做大纲」

若用户仅提供链接或长文，先总结或读取内容后再执行本流程。

---

## 执行流程（5 步）

### 第一步：理解材料

- 读取用户提供的文档、粘贴内容、本地文件路径或链接内容。
- 概括：主题是什么、目标读者/难度、是否已有小节或标题。
- **出口条件**：能一句话说明「讲什么、给谁、深度如何」。

### 第二步：确定元信息

- 与用户确认或合理推断：课程名称、目标人群、建议总学时、前置知识；可补充 3–5 条学习目标。
- 缺失时用「待确认」标出，并在输出末尾提示用户补全。
- **出口条件**：元信息完整或已标注待确认。

### 第三步：拆章节

- 按逻辑顺序拆成章（必要时再拆节），每章/节有标题和一句话描述。
- 粒度建议：一章约 30–60 分钟学习量，一节约 10–20 分钟。
- **出口条件**：章节无重叠、顺序合理、标题与描述清晰。

### 第四步：填知识点与练习建议

- 每节下列出 3–7 个知识点，标 [掌握] 或 [了解]；避免跨节重复、遗漏该节核心内容。
- 每节或每章给出练习形式（习题/项目/讨论/实操）及数量或时长建议。
- **出口条件**：每节知识点完整、层级一致，且至少 1 条练习建议。

### 第五步：输出与校验

- 按下方「输出格式」生成一份完整大纲；自检层级与遗漏。
- **出口条件**：符合模板、可直接使用或保存。

---

## 输出格式

必须按以下结构输出，缺少的元信息用「待确认」标出。若用户只要大纲/目录，可仅输出元信息 + 章节目录，章内不展开知识点与练习，后续按章补充。

```markdown
# [课程名称]

## 元信息
- **目标人群**：（如：初中生 / 职场新人 / 有编程基础者）
- **建议学时**：（如：共 8 小时 / 4 周，每周 2 小时）
- **前置知识**：（如：无 / 高中数学 / 会使用终端）
- **学习目标**：（3–5 条）

## 章节目录
1. [第一章标题] — 一句话描述
2. [第二章标题] — 一句话描述
...

## 第一章：[标题]
### 描述
本节在整体中的位置、与前后章的关系（1–2 句）。

### 知识点
- [掌握] 知识点 A：简要说明
- [了解] 知识点 B：简要说明

### 练习建议
- 类型（选择/填空/项目/讨论）：数量或形式

## 第二章、第三章：…
（结构同上，每章含 描述 / 知识点 / 练习建议）

## 附录（可选）
- 参考资料、扩展阅读
```

---

## 更新已有大纲

当用户说「在现有大纲上加一章」或「更新第 X 章」时：

1. 读取已有大纲文件或用户指定的文档。
2. 定位要新增或修改的章节。
3. 仅变更对应部分，再输出完整文档。
4. 若用户提供的是已保存的大纲文件，更新后提醒用户保存或覆盖原文件。

---

## 与其他 Skill 的衔接

- **作业图片识别与解答**：大纲中的题目/练习可后续用该 Skill 做识别与解答。
- **课程表导入与日历提醒**：可按大纲章节或学时生成课表事件并导入日历。

---

## 输出约定

- 先给材料理解结论（主题、章数、节数、建议学时），再按模板输出完整大纲；有「待确认」时在文末列出。
- 全部完成后给出大纲摘要（章数、总节数、建议学时），并说明可保存位置。

## 验证标准

- 大纲包含元信息、章节目录、每章有描述 + 知识点 + 练习建议，层级清晰，可直接用于学习或二次编辑。
