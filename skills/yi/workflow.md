# 周易智慧工作流

本文件定义各使用场景的完整执行流程。根据用户意图选择对应流程。

## 场景识别

收到用户请求后，先判断属于哪个场景：

| 场景 | 识别信号 | 是否调脚本 |
|------|----------|-----------|
| 起卦占卜 | 「起卦」「占卜」「帮我算一卦」 | 是 |
| 卦象查询 | 「查询 X 卦」「第 N 卦」 | 否 |
| 卦象解读 | 「解读」「这个卦什么意思」 | 否 |
| 易经哲学 | 「阴阳」「三才」「易经思想」 | 否 |
| 决策参考 | 「帮我分析」「该怎么选」+ 易经语境 | 可能 |

---

## 流程一：起卦占卜

最完整的流程，涉及脚本调用和多步解读。

### Step 1：明确问题

- 了解用户想占问的具体事项
- 如果用户没有明确问题，引导其聚焦：「你最想了解哪方面的情况？事业、感情、决策？」
- 确认起卦方式偏好（默认铜钱起卦）

**退出条件**：用户问题已明确，起卦方式已确定。

### Step 2：执行起卦

根据用户选择的方式调用脚本：

```python
import os, subprocess, sys, json

skill_dir = r"<本技能绝对路径>"
script = os.path.join(skill_dir, "scripts", "divination.py")

# 铜钱起卦
result = subprocess.run(
    [sys.executable, script, "coin"],
    cwd=skill_dir, capture_output=True, text=True, timeout=120,
)

# 数字起卦：[sys.executable, script, "number", str(n1), str(n2)]
# 时间起卦：[sys.executable, script, "time"]
```

**退出条件**：脚本返回退出码 0，获得 JSON 结果。

**异常处理**：
- 脚本执行失败 → 检查 stderr 信息，告知用户并尝试修复
- 超时 → 重试一次，仍失败则用备用方式（随机数模拟）

### Step 3：呈现卦象

向用户展示起卦结果，格式：

```text
━━━ 起卦结果 ━━━

起卦方式：[铜钱/数字/时间]
本　　卦：[卦名]（[上卦名][下卦名]）[卦象符号]
变　　爻：第 [N] 爻
变　　卦：[卦名]（[上卦名][下卦名]）[卦象符号]

六爻（从下到上）：
初爻：[阴/阳] [是否变爻]
二爻：[阴/阳] [是否变爻]
三爻：[阴/阳] [是否变爻]
四爻：[阴/阳] [是否变爻]
五爻：[阴/阳] [是否变爻]
上爻：[阴/阳] [是否变爻]
```

**退出条件**：卦象信息完整呈现。

### Step 4：深度解读

按以下顺序进行解读：

1. **卦辞总断**：本卦的整体含义
2. **象辞启示**：君子应如何效法
3. **变爻分析**：动爻的具体指引（按变爻规则，见 interpretation-guide.md）
4. **变卦趋势**：事态的发展方向
5. **综合判断**：结合用户问题给出针对性分析

**解读要求**：
- 先引用原文，再用现代语言解释
- 结合用户的具体问题，不做泛泛而谈
- 吉凶判断说明依据
- 给出具体可行的建议

**退出条件**：五个层次的解读全部完成。

### Step 5：总结建议

- 用 2-3 句话概括核心启示
- 给出具体的行动建议
- 提醒「易经是智慧参考，最终决策在于自身」

**退出条件**：用户收到完整的解读和建议。

## 流程二：卦象查询

用户想了解某个特定卦的信息。

### Step 1：识别目标卦

- 按卦名查：「乾卦」「坤卦」
- 按序号查：「第三卦」「第 15 卦」
- 按卦象查：「天地否」「水火既济」

### Step 2：呈现完整信息

```text
━━━ [卦名]（第 [N] 卦）━━━

卦象：[上卦符号][下卦符号]（上[上卦名]下[下卦名]）

【卦辞】[原文]

【彖辞】[原文]

【象辞】[原文]

【六爻】
初[九/六]：[爻辞]
[六/九]二：[爻辞]
[六/九]三：[爻辞]
[六/九]四：[爻辞]
[六/九]五：[爻辞]
上[九/六]：[爻辞]

【核心含义】[现代语言解释]
```

### Step 3：延伸说明（如用户有兴趣）

- 此卦在序卦中的位置和逻辑
- 与相关卦的关系（错卦、综卦、互卦）
- 历史典故或应用案例

## 流程三：卦象解读

用户已有卦象（自己起的或之前起的），需要解读。

### Step 1：确认卦象信息

需要确认：
- 本卦是什么？
- 有哪些变爻？
- 变卦是什么？
- 用户占问的问题是什么？

如果信息不完整，逐一询问。

### Step 2：多维度解读

按 interpretation-guide.md 中的方法进行解读：
1. 本卦整体分析
2. 变爻重点分析（按变爻数量选择规则）
3. 变卦趋势分析
4. 互卦补充分析（如需要）

### Step 3：结合问题给出建议

将卦象含义与用户问题对接，给出具体建议。

## 流程四：易经哲学

用户想了解易经的哲学思想。

### 执行方式

- 根据用户问题，从 philosophy.md 中选取相关内容
- 用通俗语言解释，辅以卦象实例
- 可结合现代生活场景说明应用价值

### 常见话题

| 话题 | 核心要点 |
|------|----------|
| 阴阳 | 互根、消长、转化、平衡 |
| 三才 | 天时、地利、人和 |
| 时位 | 六爻位置与人生阶段 |
| 变易 | 唯一不变的是变化 |
| 中正 | 二五之位，守中守正 |

---

## 流程五：决策参考

用户面临选择，希望借助易经智慧分析。

### Step 1：了解决策背景

- 面临什么选择？
- 有哪些选项？
- 主要的顾虑是什么？

### Step 2：判断是否需要起卦

- 如果用户希望起卦 → 转入流程一
- 如果用户只想借助易经思维分析 → 直接进入 Step 3

### Step 3：易经框架分析

用易经的思维框架分析决策：

| 分析维度 | 对应易理 |
|----------|----------|
| 当前形势 | 对应哪个卦的状态？ |
| 变化趋势 | 事态在往哪个方向发展？ |
| 时机判断 | 是「潜龙勿用」还是「飞龙在天」？ |
| 风险评估 | 有哪些「亢龙有悔」的风险？ |
| 行动建议 | 应刚应柔？应进应退？ |

### Step 4：给出建议

- 综合分析给出倾向性建议
- 说明依据和逻辑
- 强调最终决策在于用户自身

---

## 质量检查

每次交互完成前，检查：

- [ ] 引用的卦辞、爻辞是否准确
- [ ] 解读是否结合了用户的具体问题
- [ ] 是否避免了武断的吉凶判断
- [ ] 是否提醒了「易经是参考，不是预测」
- [ ] 语言是否庄重而不晦涩
