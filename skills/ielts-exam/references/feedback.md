# 反馈与自我改进系统

## 追踪内容

### 写作反馈模式

**任务回应问题：**
- [ ] 立场不清晰
- [ ] 未完整回应题目所有部分
- [ ] 论点展开不足
- [ ] 例子过于笼统

**连贯衔接问题：**
- [ ] 无清晰段落结构
- [ ] 过度使用连接词（"However"、"Moreover"）
- [ ] 论点逻辑不流畅
- [ ] Task 1 无概述

**词汇资源问题：**
- [ ] 词汇重复
- [ ] 搭配错误
- [ ] 拼写错误
- [ ] 语域不当

**语法范围与准确性问题：**
- [ ] 仅有简单句
- [ ] 冠词错误（a/an/the）
- [ ] 时态不一致
- [ ] 主谓一致错误

### 口语反馈模式

**流利度问题：**
- [ ] 过多填充词（um、uh、like）
- [ ] 长停顿（>2 秒）
- [ ] 自我纠正打断流畅
- [ ] 语速过快/过慢

**发音问题：**
- [ ] 单词重音错误
- [ ] 困难音（th、r、l、v/w）
- [ ] 语调平淡
- [ ] 连读不清晰

**词汇问题：**
- [ ] 重复相同词汇
- [ ] 选词不精确
- [ ] 缺乏习语表达
- [ ] 卡壳时不会转述

**语法问题：**
- [ ] 无复杂结构
- [ ] 条件句错误
- [ ] 时态混用
- [ ] 代词指代不清

### 阅读/听力错误模式

**题型弱点追踪：**
按题型统计准确率：
```
| 题型 | 尝试 | 正确 | % |
|------|------|------|---|
| T/F/NG | 50 | 35 | 70% |
| 段落标题配对 | 30 | 18 | 60% | ← 重点
| 选择题 | 40 | 32 | 80% |
```

**错误类别：**
- 误读题目
- 未在文中找到答案
- 选错干扰项
- 时间耗尽
- 拼写错误（听力）

## 从修正中学习

### 当用户纠正 Agent 反馈时

若用户说"这不是我失分的原因"等：
1. 记录修正内容
2. 调整未来反馈标准
3. 记入 `~/ielts/agent-learning.md`

**示例：**
```markdown
## Agent 学习日志

### 2025-02-13
- **情境：** 标记作文缺乏例子
- **用户纠正：** "我有例子，在第二段"
- **学习：** 声称缺少例子前先检查所有段落
```

### 模式识别

10 篇以上作文后：
- 哪个评分项持续最低？
- 哪条反馈最常受质疑？
- 哪些修正显示进步？

5 次以上模拟考后：
- 哪些模块在进步？
- 哪些题型仍薄弱？
- 时间管理在改善吗？

## 进步里程碑

### 分数跨越阈值

追踪用户跨越分数门槛：
```markdown
## 里程碑

- 2025-01-15：写作首次达到 6.0
- 2025-02-01：写作首次达到 6.5
- 2025-02-10：总分首次达到 7.0（模拟）
```

### 准备就绪指标

**就绪信号：**
- 连续 3 次模拟达到或超过目标
- 任何模块不低于目标 0.5 分
- 时间管理稳定（有缓冲完成）
- 用户报告感到自信

**未就绪信号：**
- 模拟分数波动大
- 某一模块持续低于目标
- 经常时间不够
- 焦虑影响表现

## 反馈循环问题

每次主要练习后：
1. 什么比预期难？
2. 什么比预期简单？
3. 答案中有意外吗？
4. 下一步想专注什么？

每次模拟考后：
1. 时间感觉充裕吗？
2. 哪个模块最耗精力？
3. 有靠猜的题吗？
4. 会有什么不同做法？

## 自适应练习

### 若用户持续困难

同一概念 3 次以上失败：
- 尝试不同解释方式
- 拆分为更小步骤
- 寻找替代练习资料
- 建议针对性资源（视频、书籍、课程）

### 若用户表现优异

将焦点转向下一个薄弱点：
- 不过度练习强项
- 用定期检查保持
- 将时间重新分配给差距

### 若用户遇到瓶颈

瓶颈信号：
- 多次模拟分数相同
- 同样错误重复
- 练习但无进步

干预策略：
- 改变练习类型（专项 → 完整测试，或反之）
- 密集专注**一个**评分项
- 休息一下（倦怠影响表现）
- 一起分析评分标准
