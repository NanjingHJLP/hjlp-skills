# 研究报告模板

本文件提供三种报告模板，根据研究类型选择使用。所有模板均可根据具体主题调整小节名称，但整体结构应尽量保留。

---

## 模板一：标准研究报告

适用于：概念理解、技术调研、综合调研。

```markdown
# [研究主题] 研究报告

> 研究日期：[YYYY-MM-DD]
> 研究范围：[时间范围、地域范围、深度说明]

## 概述

[用 2-3 句话总结核心发现和最重要的结论。读者只看这一段就能抓住要点。]

## 详细分析

### [维度 1 标题]

[分析内容。每个关键论点附带来源。]

> 关键数据：[具体数字]（来源：[标题](URL)，[时间]）

### [维度 2 标题]

[分析内容。]

### [维度 3 标题]

[分析内容。如有争议，呈现各方观点：]

> **观点 A**：[内容]（来源：[标题](URL)）
> **观点 B**：[内容]（来源：[标题](URL)）
> **分析**：[你的判断和依据]

## 关键发现

1. [发现 1]（来源：[标题](URL)）
2. [发现 2]（来源：[标题](URL)）
3. [发现 3]（来源：[标题](URL)）

## 局限与说明

- [本次研究的局限性，如某方面数据不足]
- [未验证信息的说明]
- [时效性提醒]

## 参考来源

1. [来源标题](URL) — [一句话摘要]（[发布日期]）
2. [来源标题](URL) — [一句话摘要]（[发布日期]）
3. [来源标题](URL) — [一句话摘要]（[发布日期]）
```

## 写作规范

### 引用格式

行内引用：
```
根据 [来源标题](URL) 的数据，[具体内容]。
```

数据引用：
```
> [具体数据]（来源：[标题](URL)，[数据时间]）
```

争议标注：
```
> **注意**：该数据在不同来源中存在差异。[来源 A](URL) 报告为 X，
> 而 [来源 B](URL) 报告为 Y。本报告采用 [选择]，原因是 [理由]。
```

### 语言要求

- 报告语言跟随用户语言
- 专业术语首次出现时给出解释或英文原文
- 避免口语化表达，保持客观中立
- 结论用肯定句，推测用「可能」「预计」等限定词

### 结构要求

- 概述必须能独立传达核心发现
- 每个大节控制在合理长度，过长则拆分子节
- 表格用于呈现结构化数据和对比
- 列表用于枚举要点

### 禁止事项

- 不要把搜索结果逐条罗列成流水账
- 不要只引用单一来源形成强结论
- 不要省略时间信息
- 不要遗漏参考来源列表
- 不要在报告中使用「我搜索了」「我发现了」等第一人称叙述
