# 智能排程算法详解

## 概述

排程引擎采用多约束优化算法，综合考虑时间约束、任务优先级、用户偏好等因素，生成最优学习排程。

## 核心算法

### 1. 任务优先级计算（Eisenhower矩阵）

```python
priority = urgency * 0.4 + importance * 0.4 + difficulty * 0.2
```

#### 紧急度评分（Urgency）

| 截止时间 | 紧急度 |
|---------|-------|
| ≤24小时 | 10 |
| ≤72小时 | 8 |
| ≤1周 | 6 |
| >1周 | 4 |
| 无截止 | 5 |

#### 重要度评分（Importance）

用户指定 1-10，默认5分。

#### 难度系数（Difficulty）

| 难度 | 系数 |
|-----|------|
| easy | 1 |
| medium | 2 |
| hard | 3 |

### 2. 时间段分配算法

#### 2.1 可用时间段识别

```
输入：固定事件（课程）、每日时间范围
输出：空闲时间段列表

步骤：
1. 遍历每一天（周一到周日）
2. 获取当天的固定事件，按时间排序
3. 计算事件间的空闲间隙
4. 过滤小于最小时间块的间隙
5. 返回所有可用时间段
```

#### 2.2 最佳时间段选择

评分因素（总分越高越优先）：

| 因素 | 加分条件 | 分值 |
|-----|---------|------|
| 偏好时段 | 晚间偏好且时段在19-22点 | +20 |
| 偏好时段 | 早间偏好且时段在8-11点 | +20 |
| 黄金时间 | 困难任务+高效时段 | +15 |
| 时间匹配 | 时间段能容纳完整任务 | +10 |
| 截止日期 | 截止前1天 | +25 |
| 截止日期 | 截止前3天 | +15 |
| 连续性 | 同科目分散加分 | +5 |

### 3. 约束处理

#### 3.1 硬约束（必须满足）

1. **不可重叠**：固定事件（课程）之间不能冲突
2. **时间边界**：所有事件必须在 day_start ~ day_end 范围内
3. **最小时间块**：学习任务不能小于 min_slot_size（默认30分钟）
4. **休息间隔**：强制插入 break_duration（默认15分钟）

#### 3.2 软约束（尽量满足）

1. **偏好时段**：尽量安排在用户高效时段
2. **科目连续性**：同科目建议集中学习
3. **疲劳度控制**：避免连续学习超过 max_continuous_study（默认2小时）
4. **负荷均衡**：避免单日任务过多（>10小时）

### 4. 冲突解决策略

当检测到冲突时，按以下优先级解决：

1. **课程优先**：固定课程不可调整
2. **截止时间优先**：紧急任务优先安排
3. **时段调整**：尝试将冲突任务移动到其他时间段
4. **任务拆分**：将大任务拆分为多个小任务
5. **延后处理**：非紧急任务延后安排

## 算法流程

```
输入：固定事件列表、任务列表、用户偏好、周开始日期
输出：完整排程表

1. 计算每个任务的优先级
2. 识别所有可用时间段
3. 按优先级排序任务
4. 对每个任务：
   a. 计算剩余需分配时间
   b. 在可用时间段中选择最佳时段
   c. 分配任务片段（不超过最大连续学习时间）
   d. 更新可用时间段
   e. 添加休息间隔
5. 合并固定事件和学习事件
6. 检查冲突
7. 返回排程表
```

## 优化策略

### 1. 遗传算法优化（可选）

对于复杂场景，可使用遗传算法进一步优化：

```python
# 伪代码
def genetic_optimize(schedule):
    population = [schedule] * 100  # 初始种群
    
    for generation in range(100):
        # 评估适应度
        fitness = [evaluate(s) for s in population]
        
        # 选择
        parents = select(population, fitness)
        
        # 交叉
        offspring = crossover(parents)
        
        # 变异
        offspring = mutate(offspring)
        
        # 更新种群
        population = offspring
    
    return best(population)

# 适应度函数
def evaluate(schedule):
    score = 0
    score += preference_match(schedule) * 0.3
    score += conflict_penalty(schedule) * -0.5  # 冲突扣分
    score += continuity_bonus(schedule) * 0.2
    return score
```

### 2. 动态调整

根据历史完成情况自动调整：

| 发现模式 | 调整策略 |
|---------|---------|
| 某类任务经常超时 | 增加预估时间20% |
| 特定时段效率高 | 困难任务优先安排此时段 |
| 连续高强度学习 | 建议增加休息 |
| 空闲时间浪费 | 推荐碎片化学习任务 |

## 复杂度分析

- **时间复杂度**：O(n*m)，n为任务数，m为时间段数
- **空间复杂度**：O(n+m)

实际应用中n<100, m<50，性能良好。
