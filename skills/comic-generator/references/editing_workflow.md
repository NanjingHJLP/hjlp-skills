# 单页编辑工作流程

## 编辑触发时机

用户可能在以下情况要求编辑：
1. 预览后不满意某页图像
2. 想修改某页文字
3. 想改变某页风格（素描/彩绘）
4. 发现错误需要修正

## 编辑流程

### 1. 识别要编辑的页面
- 展示所有页面预览（编号、缩略文字、图像状态）
- 询问用户："要编辑第几页？"
- 确认页面编号（1-based）

### 2. 确定编辑类型
询问用户：
```
要编辑什么？
1. 图像（重新生成）
2. 文字（修改描述）
3. 风格（切换素描/彩绘）
4. 全部重做
```

### 3. 执行编辑

#### 3.1 编辑图像
- 获取该页当前文本
- 询问用户是否修改 prompt（可选）
- 使用更新后的 prompt 调用 `image_generate`
- 替换原图像文件
- 更新页面状态

#### 3.2 编辑文字
- 显示当前文字
- 询问新文字
- 更新页面文本
- 询问是否重新生成图像（文字变化可能需新图像）
- 如需重新生成，跳转至 3.1

#### 3.3 编辑风格
- 切换页面风格（sketch ↔ color）
- 自动重新生成 prompt
- 询问是否立即重新生成图像
- 如需，调用 `image_generate`

#### 3.4 全部重做
- 重置该页所有内容
- 重新输入文本
- 重新选择风格
- 重新生成图像

### 4. 更新项目状态
- 标记页面为已编辑 (`edited=True`)
- 保存项目状态（JSON）
- 更新预览

## 技术实现

### 页面状态管理
使用 `ComicPage` 类跟踪：
- `original_text`：原始文本
- `current_text`：当前文本（编辑后）
- `style`：当前风格
- `image_path`：图像文件路径
- `edited`：是否编辑过

### 图像文件处理
编辑时：
1. 保留原图像作为备份（可重命名添加 `.bak`）
2. 新图像使用相同文件名覆盖，或新文件名带版本号
3. 更新 `image_path` 引用

### 用户确认
每次编辑后：
1. 显示编辑后的页面预览
2. 询问用户是否满意
3. 不满意可继续编辑或撤销

## 撤销功能

建议实现简单撤销：
- 保存最近一次编辑前的状态
- 提供"撤销"选项
- 可恢复原图像和文本

## 批量编辑

如果用户想编辑多个页面：
1. 询问页面编号列表（如"1,3,5" 或 "2-4"）
2. 针对每个页面执行编辑流程
3. 批量确认

## 示例对话

**用户**："我想修改第3页的图像，感觉不够生动"

**Agent**：
1. 显示第3页当前内容
2. 询问是否修改 prompt 或直接重新生成
3. 如用户提供新描述，构建新 prompt
4. 调用 `image_generate` 生成新图像
5. 展示新图像，确认是否满意
6. 更新项目状态

## 注意事项

1. **图像生成成本**：编辑会消耗额外生成次数，提醒用户
2. **一致性**：编辑后页面应与整体风格协调
3. **文件管理**：避免产生过多临时文件
4. **用户体验**：编辑过程应简单直观，避免冗长确认
